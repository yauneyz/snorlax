# Safari Web Extension native bridge

These files are overlaid onto the Xcode project generated by Apple's
`safari-web-extension-packager` during a macOS `build:extension`.

The Safari background script uses one-shot native messages. `SafariWebExtensionHandler.swift`
connects each message to the existing Talysman service Unix socket, returns the current blocking
state, and relays a heartbeat tagged with Safari's root process ID. This preserves the same
fail-safe DNR and browser-watchdog model used by the Chromium and Firefox native hosts.

The extension is embedded in `Talysman.app/Contents/PlugIns` by `scripts/after-pack.mjs` before
electron-builder signs and notarizes the macOS app. Windows and Linux builds never invoke the Safari
packager and never create or embed an `.appex`.

The temporary absolute-path sandbox exception is required for the Safari app extension to reach
the root-owned service socket. Retain its App Store/reviewer justification with the native-messaging
and desktop-companion explanation.
